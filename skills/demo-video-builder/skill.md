# Demo Video Builder

Turn an animated HTML page into a 1080p MP4 you can post to X, LinkedIn, Product Hunt, or your landing page.

Part of **Super Distributor** — built to close the gap between "I have a landing page" and "I have a shareable demo".

---

## What This Skill Does

Takes an animated HTML file and produces a **single MP4 video** ready for social media:
- ✅ **1080p H.264 MP4** — plays everywhere (X, LinkedIn, Product Hunt, web)
- ✅ **Three audio modes** — silent, external audio file, or Web Audio API (offline-rendered for perfect sync)
- ✅ **Deterministic** — pure Playwright + ffmpeg, no screen recording
- ✅ **Fast** — a 30-second demo renders in ~90 seconds on a modern Mac
- ✅ **No system dependencies** — ffmpeg is pulled in via `imageio-ffmpeg`, no Homebrew required

---

## When to Use This Skill

Use this skill when you have:
- An animated HTML landing page (maybe generated by the `landing-page-builder` skill)
- A self-contained HTML demo that plays an animation sequence
- A product walkthrough built as a single HTML file

Don't use this skill for:
- Screen recording of an existing web app (use OBS or macOS Screen Recording)
- Video editing, trimming, or transitions between clips (use DaVinci Resolve or ffmpeg directly)
- Video files you already have (just use ffmpeg)

pip install playwright imageio-ffmpeg

## Step 1: Prepare the HTML

The HTML must be **self-contained** (inline CSS, inline JS, no local imports). The skill supports two common patterns:

### Pattern A: Auto-playing animation
The animation starts as soon as the page loads. Nothing else needed.

### Pattern B: Click-to-start animation
The animation waits for a user gesture (common when using `AudioContext` — browsers block it otherwise). The script handles this by clicking the page center automatically. If your page starts on click, you don't need any flags.

### Figuring out duration
Open the HTML in a browser, start the animation, and time it with a stopwatch. Add 1-2 seconds of buffer for safety. This becomes your `--duration`.

## Step 2: Decide the Audio Mode

| Mode | When to use | How |
|------|-------------|-----|
| **Silent** | Music will be added later in an editor, or the visuals speak for themselves | Don't pass any audio flag |
| **External file** | You have an mp3/wav/m4a file (royalty-free track, voiceover, etc.) | `--audio-file music.mp3` |
| **Web Audio API** | Your HTML generates music via `AudioContext` at runtime | `--audio-js soundtrack.js` |

### Why Web Audio API needs special handling

Playwright's built-in video recorder captures visuals, but **not** audio generated by `AudioContext` — that output goes to the system speakers, not into the recording pipeline. The clean solution: render the same audio offline using `OfflineAudioContext`, then merge with ffmpeg.

The `--audio-js` flag lets you provide a JavaScript file with the same Web Audio scheduling logic as your HTML, but targeted at `offlineCtx` instead of `audioCtx`. The script then renders it to WAV and merges it with the video.

See `examples/soundtrack-upbeat-tech.js` for a complete reference — a 37-second upbeat tech-demo soundtrack you can drop in as-is or customize.

## Step 3: Run the Script

```bash
python3 render_video.py --html demo.html --duration 30
```

Full option list:

```
--html PATH              (required) HTML file to render
--output PATH            Output MP4 path (default: demo-video.mp4)
--duration SECONDS       Recording duration in seconds (default: 30)
--width INT              Video width in px (default: 1920)
--height INT             Video height in px (default: 1080)
--audio-file PATH        External audio file to merge (mp3/wav/m4a)
--audio-js PATH          JS file with OfflineAudioContext scheduling code
--audio-sample-rate INT  Sample rate (default: 44100)
--no-click               Don't auto-click center (use if animation auto-plays)
--wait-before-record N   Seconds to wait before clicking (default: 0.3)
--keep-tmp               Keep intermediate files for debugging
```

### Common recipes

```bash
# Silent 30-second demo at 1080p
python3 render_video.py --html demo.html --duration 30

# With external soundtrack
python3 render_video.py --html demo.html --duration 30 \
  --audio-file ~/Music/royalty-free-track.mp3

# With Web Audio API soundtrack (matches HTML's AudioContext music exactly)
python3 render_video.py --html demo.html --duration 37 \
  --audio-js examples/soundtrack-upbeat-tech.js

# 720p for faster rendering and smaller file size
python3 render_video.py --html demo.html --duration 30 \
  --width 1280 --height 720

# Square format for Instagram / LinkedIn feed
python3 render_video.py --html demo.html --duration 30 \
  --width 1080 --height 1080

# Vertical format for Reels / Shorts / TikTok
python3 render_video.py --html demo.html --duration 30 \
  --width 1080 --height 1920
```

## Step 4: Verify the Output

After rendering, sanity-check the result:

```bash
# Quick info dump (uses the bundled ffmpeg)
python3 -c "import imageio_ffmpeg, subprocess; \
  subprocess.run([imageio_ffmpeg.get_ffmpeg_exe(), '-i', 'demo-video.mp4'])"
```

Look for:
- **Duration** matches what you asked for
- **Resolution** matches (1920x1080 by default)
- **Stream #0:0 Video: h264**
- **Stream #0:1 Audio: aac** (only if you passed audio)

Then open it in QuickTime / VLC and watch the full video. Things to check:
1. Animation starts at the right moment
2. No black frames at the beginning or end
3. Audio is in sync with visuals (if you have audio)
4. The final frame isn't cut off mid-animation (if cut off, increase `--duration`)

## Troubleshooting

### "Missing dependency: playwright"
```bash
pip install playwright && playwright install chromium
```

### "Missing dependency: imageio-ffmpeg"
```bash
pip install imageio-ffmpeg
```

### Video is black / nothing animates
Your HTML probably waits for a click that never happens. The script auto-clicks center, but if you have a modal or overlay that blocks the click, add a `--no-click` flag and ensure your animation starts on page load instead.

### Audio is out of sync
- If using `--audio-file`: the external audio has its own timing that doesn't know about the animation. Either trim the audio to match, or use `--audio-js` to schedule sound relative to the animation timeline.
- If using `--audio-js`: check that your `DURATION` in the JS matches what you passed to `--duration`.

### Video ends before the animation finishes
Increase `--duration`. The script records for exactly `--duration` seconds regardless of animation state.

### Music ends before the video
Your music envelope probably fades out earlier than `--duration`. Either extend the music's fade-out time, or decrease `--duration` to match the music length.

### "ffmpeg failed"
Run with `--keep-tmp` to see the raw webm file and audio wav in `_demo_render_tmp/`. The webm should be playable in VLC — if it is, the problem is in the merge step and the full ffmpeg stderr will tell you what's wrong.

## How It Works (Technical Overview)

For curious users and contributors:

1. **Audio rendering (optional)**: If `--audio-js` is passed, the script launches headless Chromium, evaluates the user JS inside a wrapper that creates an `OfflineAudioContext`, calls `startRendering()`, encodes the resulting AudioBuffer as a 16-bit PCM WAV, and returns it as base64 over the Playwright protocol.

2. **Video recording**: Playwright's `browser.new_context(record_video_dir=...)` captures DOM-rendered frames as a webm video. The script opens the HTML, optionally clicks the center (to bypass browser autoplay policies), and waits for `--duration` seconds.

3. **Encoding**: ffmpeg (bundled via `imageio-ffmpeg`) re-encodes the webm to H.264 at CRF 18 (visually lossless), muxes in the audio track (if any) as AAC, adds `+faststart` for web playback, and trims to exactly `--duration` seconds.

The whole pipeline is deterministic — re-running produces identical output bytes modulo ffmpeg encoder timing. This matters because it means you can regenerate and re-post without "wait, was this the right version?" anxiety.
