# Grok Prompt 库

**Grok = 最了解 X 社区的 AI Agent**

不需要 20 个 prompts。你只需要 3 个核心场景。

---

## 1. 找到你的用户在哪（10 分钟）

**什么时候用：** 你不知道目标用户在 X 上聚集在哪里

**实战案例：**
想了解 indie builders 在 #buildinpublic 社区讨论什么 → 用 Grok 分析 → 10 分钟内得到完整的话题分布、用户特征、痛点总结

**Prompt：**
```
分析 X 上的 #[你的目标社区标签] 社区。

告诉我：
1. 主要讨论哪些话题？（按频率排序）
2. 典型用户是什么样的人？
3. 他们最常见的痛点和问题是什么？
4. 内容同质化的问题严重吗？有哪些被忽视的话题？
```

**替换这里：**
- `#buildinpublic` → 你的目标社区标签（#indiehackers, #SaaS, #nocode 等）

**你会得到：**
- 社区话题地图
- 用户画像
- 真实痛点列表
- 内容空白点（你的机会）

**下一步：**
基于痛点，决定你的内容策略和切入角度

---

## 2. 分析具体用户（5 分钟）

**什么时候用：** 有人和你互动了，你想知道他是谁、是否是目标用户

**实战案例：**
@AnishSukhramani 回复了你的帖子 → 用 Grok 分析 → 5 分钟内了解他的背景、职业、是否是 indie builder、当前痛点

**Prompt：**
```
分析 @[用户名] 是谁。

告诉我：
1. 基本背景（职业、教育、所在地）
2. 主要兴趣和讨论话题
3. 是否是 [你的目标用户类型，如"indie builder / solo founder"]？
4. 他当前面临的主要问题是什么？
5. 如何为他提供价值？
```

**替换这里：**
- `@AnishSukhramani` → 你想了解的用户
- `indie builder / solo founder` → 你的目标用户定义

**你会得到：**
- 完整用户画像
- 是否符合目标用户的判断
- 如何和他有价值地互动

**下一步：**
用这些洞察决定是否深入对话，以及说什么

---

## 3. 发现当下的机会（每天 5 分钟）

**什么时候用：** 你想主动找到"现在就需要帮助"的用户

**实战案例：**
搜索过去 48 小时发"刚 launch 了但没人用"的帖子 → 用 Grok 找到 10 个低互动的 launch → 回复提供价值 → 建立第一批对话

**Prompt A：找需要帮助的人**
```
搜索 X 上过去 48 小时内发布的帖子，内容包含：
- "just launched" 或 "just shipped"
- 但互动很低（<20 likes）

找出 10 个案例，告诉我：
- 他们做了什么产品
- 为什么互动低（定位不清？目标用户不对？）
- 我可以如何帮助他们
```

**Prompt B：找正在讨论你解决的问题的人**
```
搜索 X 上过去 7 天内，有人在问：
- "如何获得第一批用户"
- "不知道怎么推广产品"
- [你的产品解决的具体问题]

找出 10 个最近的帖子，我可以回复提供价值。
```

**替换这里：**
- 调整搜索关键词为你的领域
- 调整时间范围（48h = 即时机会，7天 = 模式识别）

**你会得到：**
- 10 个具体的互动机会
- 每个人的背景和需求
- 如何提供即时价值的建议

**下一步：**
回复这些帖子，先提供价值（不推销），建立对话

## 4. 验证你的想法（在做之前）

**什么时候用：** 你准备写一个 playbook / 做一个功能，想知道有没有人需要

**Prompt：**
```
搜索 X 上过去 30 天，有多少人在讨论或抱怨 [你想解决的问题]。

分析：
1. 讨论量有多少？
2. 是真实痛点还是理论问题？
3. 他们用什么语言描述这个问题？
4. 现有解决方案有哪些？他们满意吗？
```

**你会得到：**
- 需求验证（值不值得做）
- 用户语言（怎么说他们才懂）
- 竞争态势（有没有空白点）

**下一步：**
如果需求真实 + 有空白 → 做。否则 → 换方向。

## 使用技巧

**1. 具体 > 模糊**
- ❌ "分析 indie hacker 社区"
- ✅ "分析 #buildinpublic 社区"

**2. 要结构化输出**
- 加上"告诉我 1/2/3/4"，结果更容易用

**3. 时间范围很重要**
- 7 天 = 趋势
- 30 天 = 模式
- 48 小时 = 即时机会

**4. 交叉验证**
- 不要只靠一个 prompt
- 用 2-3 个角度交叉确认

**5. 记录洞察**
- 别只在脑子里
- 简单记在一个文档里
- 每周回顾，找模式

---

## 核心原则

**Grok 不是自动化工具**
它是**信号放大器** — 让你更快看清市场在说什么

**不要过度调研**
- 跑 3-5 个 prompts → 得到方向 → 立刻行动
- 调研是为了更快 ship，不是拖延的借口

**记住 Ship → Observe → Adjust**
- Grok 帮你更快 Observe
- 但你还是要 Ship 和 Adjust
